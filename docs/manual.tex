\part{Resumen Ejecutivo del Proyecto}

\section{Objetivo del sistema}

Implementar un sistema de administración completo que permita gestionar todos los contenidos de UNDC Pets desde un panel centralizado. Esto incluye la administración de publicaciones del muro comunitario, el directorio de mascotas del campus, las campañas de donación y la gestión de usuarios bloqueados. Además, se incorporó la persistencia de datos de mascotas y donaciones en Supabase, con un mecanismo de respaldo en localStorage para garantizar que la información nunca se pierda, incluso si la base de datos no está disponible.

\section{Detalles importantes}

\begin{itemize}
    \item \textbf{Correo del administrador:} \texttt{2301010130@undc.edu.pe}, definido tanto en \texttt{AuthContext.tsx} como en la tabla \texttt{admins} de Supabase. Este correo es el único que tiene acceso al panel de administración.
    \item \textbf{Identificación visual:} Cuando el administrador inicia sesión, aparece una etiqueta naranja con el texto ``Admin'' junto a su nombre en la barra superior. La pestaña ``Admin'' en la navegación solo es visible cuando \texttt{isAdmin} es verdadero.
    \item \textbf{Subida de imágenes:} El panel de mascotas (\texttt{PetsPanel}) permite cargar imágenes locales desde el ordenador, convirtiéndolas a base64 para su almacenamiento, con vista previa antes de guardar.
    \item \textbf{Persistencia:} Las mascotas y campañas de donación se almacenan en localStorage bajo las claves \texttt{undc\_pets} y \texttt{undc\_campaigns} cuando la tabla de Supabase no existe. Cuando la base de datos está disponible, los datos se sincronizan automáticamente.
    \item \textbf{Edición directa:} El administrador puede editar o eliminar mascotas directamente desde el directorio principal (al pasar el mouse sobre una tarjeta aparecen iconos flotantes) o desde el panel de administración.
    \item \textbf{Migración de base de datos:} El archivo \texttt{supabase-migration.sql} contiene todas las instrucciones necesarias para crear las tablas (\texttt{admins}, \texttt{pets}, \texttt{donation\_config}, \texttt{blocked\_users}) en Supabase.
    \item \textbf{Imagen en PetCard:} Las tarjetas usan \texttt{object-cover} para la vista previa. El modal de detalle también usa \texttt{object-cover} para que la imagen llene completamente la columna izquierda sin márgenes.
\end{itemize}

A continuación se detalla el estado de cada componente del sistema al momento de esta documentación.

\subsection{Funcionalidades completadas}

    \item Se rediseñó el modal de mascotas con un diseño de imagen a la izquierda e información a la derecha, usando \texttt{object-cover} para que la imagen ocupe toda la altura disponible.
    \item Se creó \texttt{src/lib/AuthContext.tsx}, un contexto centralizado que gestiona el estado del usuario y su rol de administrador, con el correo del administrador como valor fijo de respaldo.

\part{Manual de Usuario}

\section{Introducción}

UNDC Pets es una aplicación web desarrollada para la Universidad Nacional de Cañete. Su propósito es servir como una herramienta centralizada que permite a toda la comunidad universitaria --- estudiantes, docentes, personal administrativo y vecinos de Cañete --- involucrarse activamente en el cuidado y bienestar de las mascotas que habitan dentro del campus.

La aplicación surge de la necesidad de coordinar los esfuerzos de bienestar animal que antes existían de manera dispersa: grupos de estudiantes que alimentaban a los perros y gatos del campus, voluntarios que organizaban campañas de esterilización y donaciones que se canalizaban de forma informal. UNDC Pets unifica todo en una sola plataforma accesible desde cualquier navegador web.

El sistema ofrece cuatro funcionalidades principales. En primer lugar, funciona como un directorio interactivo de las mascotas del campus, donde cualquier persona puede conocer a Curly, Princesa, RunRun y la Gata Ingeniera, con sus historias, estado de salud y ubicación. En segundo lugar, proporciona una red social interna donde los usuarios autenticados pueden publicar fotos, comentar, dar me gusta y compartir publicaciones relacionadas con las mascotas. En tercer lugar, integra un módulo de donaciones que permite realizar aportes económicos simulados a campañas específicas, así como consultar las cuentas bancarias oficiales y los códigos QR para transferencias reales. En cuarto lugar, ofrece un panel de administración para que los encargados del proyecto puedan gestionar todo el contenido de manera centralizada.

La aplicación está construida con tecnologías web modernas como React 19, Vite 6, TypeScript y Tailwind CSS, y utiliza Supabase como backend, lo que garantiza la sincronización de datos en tiempo real entre todos los dispositivos. El despliegue se realiza en Cloudflare Pages, asegurando un acceso rápido y confiable desde cualquier ubicación.

\section{Requisitos de uso}

Para utilizar UNDC Pets en su totalidad, solo necesita tres elementos: un dispositivo con acceso a internet, un navegador web moderno (Google Chrome, Mozilla Firefox, Microsoft Edge o Safari en sus versiones más recientes) y una cuenta de Google.

La cuenta de Google es necesaria únicamente para las funciones interactivas: publicar en el muro, dar me gusta, comentar, compartir publicaciones o realizar donaciones. Si no tiene una cuenta de Google o prefiere no iniciar sesión, puede navegar sin restricciones por el directorio de mascotas, el álbum de fotos y las secciones informativas, aunque no podrá interactuar con la comunidad.

La aplicación está diseñada para funcionar tanto en computadoras de escritorio como en teléfonos móviles y tabletas. La interfaz se adapta automáticamente al tamaño de la pantalla, ofreciendo una experiencia de uso cómoda en cualquier dispositivo. En dispositivos móviles, la navegación se realiza a través de un menú desplegable que se activa con un botón de tres líneas en la esquina superior derecha.

\section{Inicio de sesión}

El proceso de inicio de sesión es sencillo y familiar para cualquier persona que haya usado servicios de Google. En la esquina superior derecha de la página, junto al botón naranja ``Apadrinar Mascota'', se encuentra el botón ``Ingresar con Google'', identificado con el logo de cuatro colores de Google.

Al hacer clic en él, el navegador redirige a la página de autenticación de Google, donde debe seleccionar la cuenta que desea utilizar y otorgar los permisos necesarios. Una vez completado el proceso, será redirigido de vuelta a la aplicación.

Cuando el inicio de sesión es exitoso, ocurren varios cambios en la interfaz. En el lugar donde antes estaba el botón ``Ingresar con Google'' aparecen su foto de perfil (o sus iniciales si no tiene foto), su nombre y un botón rojo con la palabra ``Salir''. La sesión se mantiene activa incluso si cierra el navegador y vuelve a entrar, gracias a que Supabase conserva la sesión. Para cerrar sesión, simplemente haga clic en ``Salir''.

Es importante tener en cuenta que la sesión solo se mantiene en el navegador que utilizó para iniciar sesión. Si ingresa desde otro dispositivo o navegador, deberá autenticarse nuevamente.

\section{Barra de navegación}

La barra de navegación es el elemento principal para desplazarse por la aplicación. Está ubicada en la parte superior de la página y permanece fija mientras se desplaza hacia abajo. En computadoras de escritorio, los enlaces se muestran horizontalmente. En dispositivos móviles, se ocultan detrás de un botón de menú que se despliega al hacer clic.

Los enlaces disponibles son los siguientes:

\begin{itemize}
    \item \textbf{Mascotas del Campus}: directorio principal con todas las mascotas registradas.
    \item \textbf{Comunidad}: muro de publicaciones y formulario para reportar mascotas.
    \item \textbf{Álbum}: galería de fotos de las mascotas y la comunidad.
    \item \textbf{Donaciones}: campañas activas, cuentas bancarias y códigos QR.
    \item \textbf{Usuario}: perfil personal del usuario autenticado.
    \item \textbf{FAQs}: respuestas a preguntas frecuentes.
    \item \textbf{Admin}: visible únicamente para administradores del sistema.
\end{itemize}

Además, la barra contiene el logo de UNDC Pets a la izquierda: un ícono de huella dentro de un círculo azul oscuro acompañado del nombre del proyecto. Haciendo clic en el logo regresa al directorio de mascotas desde cualquier sección.

\section{Directorio de mascotas}

La sección ``Mascotas del Campus'' es el corazón de la aplicación. Al ingresar, lo primero que ve es un encabezado visual con un mensaje de bienvenida y un carrusel automático que muestra las fotos de las mascotas más destacadas. El carrusel cambia de imagen cada pocos segundos y puede pausarse pasando el mouse sobre él.

Debajo del encabezado se encuentra el directorio con un buscador y filtros en la parte superior, seguido de una cuadrícula de tarjetas, una por cada mascota registrada. Cada tarjeta contiene una foto de la mascota, su nombre, un indicador de estado de salud con un color distintivo, su especie y género, su edad, la ubicación dentro del campus, una breve descripción y hasta tres etiquetas que la describen (como ``Juguetón'', ``Vacunado'' o ``Sociable'').

El buscador permite encontrar mascotas escribiendo su nombre, la ubicación donde suele encontrarse o alguna etiqueta. Por ejemplo, si escribe ``biblioteca'', aparecerá Princesa, que es la perrita que prefiere la tranquilidad de la biblioteca. Los filtros permiten mostrar solo perros o solo gatos, o filtrar por estado de salud como ``Buscando hogar'', ``Saludable'' o ``En tratamiento''. También hay etiquetas rápidas debajo del buscador que, al hacer clic, aplican automáticamente ese término de búsqueda.

Al hacer clic en una tarjeta de mascota, se abre un modal con la información completa. En computadoras de escritorio, el modal muestra la foto de la mascota en el lado izquierdo ocupando aproximadamente la mitad del ancho, y la información en el lado derecho. En dispositivos móviles, la foto aparece arriba y la información abajo. La información incluye una ficha con especie, género y edad; la historia completa de la mascota; una ficha médica que indica si está vacunada, desparasitada, esterilizada y con evaluación veterinaria regular; y la ubicación exacta dentro del campus.

\section{Comunidad: publicaciones e interacciones}

La sección ``Comunidad'' es el espacio donde los usuarios interactúan entre sí. Al ingresar, verá dos subpestañas: ``Muro'' y ``Reportar''.

\subsection{Muro de publicaciones}

El muro muestra todas las publicaciones creadas por los usuarios, ordenadas de la más reciente a la más antigua. Cada publicación muestra el nombre del autor, su foto de perfil o iniciales, el rol que se asignó (como ``Estudiante de Agronomía'' o ``Comunidad UNDC''), el contenido del mensaje, una imagen opcional, la cantidad de me gusta, comentarios y compartidos, y el tiempo transcurrido desde su publicación.

Para crear una publicación, debe haber iniciado sesión con su cuenta de Google. En la parte superior del muro hay un campo de texto que dice ``¿Qué quieres compartir sobre nuestras mascotas?''. Escriba su mensaje allí. Si desea agregar una imagen, puede pegar una URL en el campo que aparece al hacer clic en el ícono de imagen. Una vez listo, haga clic en ``Publicar''. Su publicación aparecerá inmediatamente en el muro; si otro usuario está viendo la página en ese momento, la verá aparecer en tiempo real gracias a Supabase Realtime.

Puede interactuar con las publicaciones de tres maneras:

\begin{itemize}
    \item \textbf{Me gusta}: haga clic en el ícono de corazón debajo de la publicación. El corazón se llenará de color y el contador aumentará.
    \item \textbf{Comentar}: escriba su mensaje en el campo de comentarios y presione Enter. Su comentario aparecerá inmediatamente.
    \item \textbf{Compartir}: haga clic en el botón de compartir para que la publicación aparezca en la pestaña ``Compartidos'' de su perfil.
\end{itemize}

Si la publicación es suya, verá botones adicionales para editarla o eliminarla al pasar el mouse sobre ella. Al hacer clic en editar, el contenido se vuelve editable. Al hacer clic en eliminar, se le pedirá confirmación antes de borrarla definitivamente.

\subsection{Reportar una mascota}

La subpestaña ``Reportar'' está diseñada para situaciones en las que alguien encuentra una mascota perdida, herida o en situación de riesgo dentro del campus. El formulario solicita el nombre de la mascota (si se conoce), la especie, el género, la edad aproximada, el estado en que se encuentra, la ubicación donde fue vista, una descripción detallada de la situación y una foto si está disponible. Al enviar el reporte, la mascota se agrega automáticamente al directorio principal para que otros usuarios puedan verla y ayudar.

\section{Álbum de fotos}

La sección ``Álbum'' combina en una sola galería todas las fotos disponibles de las mascotas del campus junto con las imágenes que los usuarios han compartido en sus publicaciones. La galería se presenta en una cuadrícula de tamaño uniforme, y al hacer clic en cualquier foto se abre una vista ampliada para apreciarla en detalle. Esta sección no requiere autenticación, por lo que cualquier visitante puede disfrutar de las fotos.

\section{Donaciones}

La sección ``Donaciones'' está diseñada para facilitar el apoyo económico al proyecto. Se divide en varias partes.

\subsection{Campañas activas}

Al ingresar, lo primero que verá son las campañas activas representadas como tarjetas con barras de progreso. Cada campaña tiene un título que indica su propósito (como ``Alimento Mensual para el Campus'' o ``Cirugía y Terapia de Firulais''), una descripción, el monto recaudado hasta el momento, la meta que se necesita alcanzar y un indicador de urgencia (Alta, Crítica o Media). La barra de progreso se llena proporcionalmente al avance de la recaudación y cambia de color cuando se alcanza la meta.

\subsection{Cómo donar con tarjeta}

Para realizar una donación simulada, haga clic en ``Donar con Tarjeta'' en la campaña de su elección. Se abrirá un formulario donde debe ingresar su nombre, su correo electrónico, el monto que desea donar (puede elegir entre montos predefinidos de 10, 20, 50 o 100 soles, o ingresar un monto personalizado) y los datos de una tarjeta de prueba. Es importante aclarar que la pasarela de pago es completamente simulada: no se realizan cobros reales ni se almacenan datos financieros. Al confirmar, verá una animación de procesamiento y luego un comprobante de donación con un código único.

\subsection{Transferencia bancaria y códigos QR}

Debajo de las campañas activas se encuentran las cuentas bancarias oficiales del proyecto. Cada cuenta muestra el nombre del banco, el número de cuenta y el CCI correspondiente. Puede copiar cualquiera de estos datos al portapapeles haciendo clic en ``Copiar''. Las cuentas disponibles incluyen el Banco de la Nación, el BCP y Yape/Plin.

Más abajo se encuentran los códigos QR para Yape, Plin, BCP y Tunqui. Estos códigos pueden ser escaneados con su aplicación bancaria para realizar transferencias rápidas desde su teléfono.

\subsection{Puntos de acopio físicos}

Además de las donaciones económicas, la sección muestra los puntos de acopio donde se pueden entregar alimentos, medicinas y otros insumos: la Puerta Principal del Campus y la Oficina de Bienestar Universitario, con sus respectivos horarios de atención.

\section{Perfil de usuario}

La sección ``Usuario'' muestra su perfil personal cuando está autenticado. Si aún no ha iniciado sesión, la sección le invitará a hacerlo.

\subsection{Su perfil}

Al iniciar sesión e ir a ``Usuario'', verá su foto de perfil de Google (o sus iniciales), su nombre completo, una descripción que puede editar haciendo clic en el ícono de lápiz, y estadísticas que incluyen la cantidad de fotos publicadas y los me gusta recibidos en total.

\subsection{Pestaña Fotos}

En la subpestaña ``Fotos'' se muestran todas las publicaciones que usted ha creado y que incluyen una imagen. Cada foto se presenta en una cuadrícula. Al hacer clic en una foto, se abre un modal que muestra la imagen en grande a la izquierda y los detalles de la publicación a la derecha, incluyendo el contenido, los me gusta y los comentarios.

\subsection{Pestaña Compartidos}

En la subpestaña ``Compartidos'' aparecen las publicaciones de otros usuarios que usted ha compartido desde el muro de la comunidad. Esto le permite guardar y tener acceso rápido a las publicaciones que le parecieron interesantes.

\subsection{Ver perfil de otro usuario}

Si en el muro de la comunidad hace clic en el nombre o la foto de otro usuario, será redirigido al perfil de esa persona, donde podrá ver su nombre, su foto y las publicaciones que ha creado. Desde allí puede regresar al muro usando el botón ``Volver a Comunidad''.

\section{Preguntas frecuentes}

La sección ``FAQs'' contiene respuestas a las preguntas más comunes sobre el proyecto. Las preguntas están organizadas en acordeones: haga clic en una pregunta para expandir su respuesta y haga clic nuevamente para contraerla. Los temas incluyen cómo funciona la donación, si personas externas a la UNDC pueden donar, si se pueden donar medicinas y los horarios de los paseos de las mascotas.

\part{Manual de Administrador}

\section{¿Qué significa ser administrador?}

El administrador de UNDC Pets es la persona encargada de supervisar y gestionar el contenido de la plataforma. A diferencia de un usuario normal, que solo puede crear, editar y eliminar sus propias publicaciones, el administrador tiene la capacidad de moderar todo el contenido generado por la comunidad y gestionar la información estructural del sitio, como el directorio de mascotas y la configuración de donaciones.

Para que una persona sea reconocida como administrador, su correo electrónico debe estar registrado en la tabla \texttt{admins} de la base de datos Supabase. Esto se realiza ejecutando un comando SQL o insertando manualmente el correo en la tabla. Actualmente, el administrador identificado es \texttt{2301010130@undc.edu.pe}.

\subsection{¿Cómo saber si soy administrador?}

Cuando un administrador inicia sesión, ocurren dos cambios visibles en la interfaz. El primero es que aparece una etiqueta naranja con el texto ``Admin'' al lado de su nombre en la barra superior, junto a su foto de perfil. El segundo es que aparece una pestaña adicional en la barra de navegación llamada ``Admin'', representada con un ícono de escudo. Esta pestaña solo es visible para el administrador; ningún otro usuario puede verla.

\section{Panel de administración}

Al hacer clic en la pestaña ``Admin'', se abre el panel de administración dividido en cuatro secciones que se alternan mediante pestañas secundarias: Posts, Mascotas, Donaciones y Bloqueados.

\subsection{Gestión de Posts}

La sección de Posts muestra todas las publicaciones de la comunidad, ordenadas de la más reciente a la más antigua. Cada publicación se presenta en una tarjeta compacta con el nombre del autor, su rol, el contenido y la cantidad de me gusta y comentarios.

Como administrador, dispone de dos acciones para cada publicación. Puede eliminar cualquier publicación haciendo clic en el ícono de papelera, independientemente de quién sea el autor. Esto es útil para retirar contenido inapropiado, spam o publicaciones que no cumplen con las normas de la comunidad. También puede bloquear al usuario que creó la publicación haciendo clic en el ícono de bloqueo. El sistema le solicitará un motivo para el bloqueo. Una vez confirmado, ese usuario no podrá crear nuevas publicaciones, aunque sus publicaciones existentes permanecerán visibles a menos que las elimine manualmente.

\subsection{Gestión de Mascotas}

La sección de Mascotas es la más utilizada del panel de administración. Aquí puede ver una lista de todas las mascotas registradas con su foto, nombre, especie, edad y ubicación.

Para agregar una nueva mascota, haga clic en ``Nueva Mascota''. Se abrirá un formulario completo con todos los campos necesarios: nombre, especie (perro o gato), género, edad, estado de salud (``Sano'', ``En tratamiento'' o ``Buscando hogar''), ubicación dentro del campus, etiquetas descriptivas separadas por coma, descripción corta para la tarjeta del directorio e historia completa de la mascota. Para la foto, puede seleccionar un archivo de imagen desde su computadora; el sistema mostrará una vista previa antes de guardar.

Para editar una mascota existente, haga clic en ``Editar'' junto a la mascota en la lista. El formulario se abrirá con los datos actuales precargados. Para eliminar una mascota, haga clic en ``Eliminar'' y la mascota se borrará inmediatamente del directorio.

Además de gestionar las mascotas desde el panel, puede editarlas o eliminarlas directamente desde el directorio principal de ``Mascotas del Campus''. Al pasar el mouse sobre cualquier tarjeta de mascota, aparecen dos iconos flotantes en la esquina superior derecha: un lápiz para editar y un tacho para eliminar. Esto permite una gestión rápida sin tener que acceder al panel de administración.

\subsection{Gestión de Donaciones}

La sección de Donaciones del panel permite modificar toda la configuración relacionada con las donaciones. Está dividida en cuatro bloques.

El primer bloque corresponde a las cuentas bancarias. Puede ver las cuentas registradas con su nombre de banco, número y CCI. Cada campo es editable directamente; también puede eliminar una cuenta o agregar una nueva haciendo clic en ``Agregar cuenta''.

El segundo bloque corresponde a los números de Yape y Plin, con dos campos de texto actualizables.

El tercer bloque corresponde a las rutas de los códigos QR. Cada servicio (Yape, Plin, BCP, Tunqui) tiene un campo para especificar la ruta de la imagen del código QR, permitiendo cambiar las imágenes cuando sea necesario.

El cuarto bloque corresponde a las campañas de recaudación. Puede ver la lista de campañas activas con su título, montos y nivel de urgencia. Para agregar una nueva campaña, haga clic en ``Nueva'' y complete el formulario. Para editar o eliminar, use los botones correspondientes.

\subsection{Gestión de Usuarios Bloqueados}

La sección de Bloqueados muestra una lista de todos los usuarios que han sido bloqueados para publicar en la comunidad. Cada entrada muestra el ID del usuario (correspondiente al ID de su cuenta de Google), el motivo del bloqueo y la fecha en que fue bloqueado.

Desde esta sección puede desbloquear a un usuario haciendo clic en ``Desbloquear'', lo que le permitirá volver a publicar en el muro. También puede bloquear a un usuario manualmente ingresando su ID de Google junto con el motivo del bloqueo.

\section{Recomendaciones para el administrador}

Se recomienda revisar el muro de la comunidad periódicamente para asegurarse de que el contenido sea apropiado y esté relacionado con el bienestar animal. Las publicaciones con spam, lenguaje inapropiado o contenido no relacionado deben ser eliminadas. Antes de bloquear a un usuario, considere si una advertencia sería suficiente; el bloqueo debe reservarse para casos reincidentes o de falta grave.

En cuanto a las mascotas, es importante mantener su información actualizada, especialmente el estado de salud y la ubicación. Si una mascota es adoptada o deja de frecuentar el campus, debe actualizar o eliminar su registro. Las fotos deben ser claras y recientes.

Respecto a las donaciones, verifique periódicamente que los números de cuenta y los códigos QR sean correctos. Si cambia el coordinador del proyecto, actualice el número de Yape y Plin. Las campañas de recaudación deben reflejar las necesidades reales del momento.

\section{Visión general de la arquitectura}

UNDC Pets es una aplicación de una sola página (SPA) construida con React 19. Esto significa que toda la interfaz de usuario se carga una sola vez en el navegador, y las interacciones posteriores no requieren recargar la página completa, sino que solo se actualizan las partes necesarias. Esto proporciona una experiencia de usuario fluida y rápida, similar a la de una aplicación de escritorio.

La aplicación se comunica con Supabase, que actúa como backend y proporciona tres servicios fundamentales: autenticación mediante Google OAuth, una base de datos PostgreSQL para almacenar toda la información y un sistema de tiempo real (Realtime) que permite que los cambios se reflejen instantáneamente en todos los dispositivos conectados.

El frontend está escrito en TypeScript, un superconjunto de JavaScript que añade tipos estáticos. Esto permite detectar errores durante el desarrollo, antes de que la aplicación llegue al usuario, resultando en un código más robusto y fácil de mantener.

Los estilos se manejan con Tailwind CSS, un framework que permite aplicar clases directamente en el HTML para definir colores, tamaños, márgenes y otros aspectos visuales, acelerando el desarrollo y manteniendo la consistencia visual en toda la aplicación.

El empaquetado del código se realiza con Vite, una herramienta moderna que reemplaza a Webpack. Vite es significativamente más rápido porque aprovecha los módulos nativos de JavaScript del navegador durante el desarrollo y produce archivos optimizados para producción.

El despliegue se realiza en Cloudflare Pages, una plataforma de alojamiento que distribuye la aplicación en una red global de servidores (CDN), garantizando tiempos de carga rápidos independientemente de la ubicación geográfica del usuario.

\section{Tecnologías utilizadas}

\subsection{React 19}

React es una biblioteca de JavaScript para construir interfaces de usuario. Su idea fundamental es que la interfaz se describe en función del estado de la aplicación: cuando el estado cambia, React actualiza automáticamente las partes de la pantalla que dependen de ese estado, eliminando la necesidad de manipular manualmente el DOM.

En UNDC Pets, React se utiliza para cada elemento visual: tarjetas de mascotas, publicaciones, modales, formularios y la barra de navegación. Cada elemento es un componente de React que puede recibir datos (props) y mantener su propio estado interno. Las características más utilizadas incluyen hooks como \texttt{useState} para el estado local, \texttt{useEffect} para efectos secundarios y \texttt{useCallback} para optimización de rendimiento.

\subsection{Vite 6}

Vite es la herramienta encargada de preparar el código para el navegador. Durante el desarrollo, inicia un servidor local que sirve los archivos de manera eficiente, aplicando reemplazo de módulos en caliente (HMR). Esto significa que al modificar un archivo y guardarlo, los cambios se reflejan en el navegador al instante, sin necesidad de recargar la página manualmente.

Para producción, Vite empaqueta todo el código: combina los archivos JavaScript, optimiza las imágenes, minimiza el CSS y genera los archivos finales en la carpeta \texttt{dist}. El resultado son archivos pequeños y eficientes que se cargan rápidamente en el navegador.

\subsection{TypeScript}

TypeScript añade tipos a JavaScript, permitiendo especificar que una variable debe cumplir con una interfaz determinada. Si en alguna parte del código se intenta acceder a una propiedad que no existe o asignar un valor de tipo incorrecto, TypeScript muestra un error durante el desarrollo.

En UNDC Pets, TypeScript se utiliza para definir todas las estructuras de datos: \texttt{Pet}, \texttt{Post}, \texttt{Comment}, \texttt{DonationConfig}, entre otras. Esto hace que el código sea más fácil de entender y modificar, porque siempre está claro qué forma tienen los datos.

\subsection{Tailwind CSS}

Tailwind CSS es un framework de estilos que proporciona cientos de clases predefinidas. Por ejemplo, en lugar de escribir un archivo CSS personalizado, se aplican clases directamente en el HTML. Una ventaja importante es que, al usar las clases directamente en el HTML, no hay riesgo de que los estilos de una parte de la página afecten accidentalmente a otra. Además, Tailwind incluye un sistema de diseño responsivo que permite especificar estilos diferentes para distintos tamaños de pantalla usando prefijos como \texttt{md:} para tabletas o \texttt{lg:} para escritorio.

\subsection{Supabase}

Supabase es el backend de UNDC Pets. Se describe como una alternativa de código abierto a Firebase y proporciona tres servicios que la aplicación utiliza extensivamente.

El servicio de autenticación maneja todo el flujo de inicio de sesión con Google. Cuando un usuario hace clic en ``Ingresar con Google'', Supabase genera una URL de redirección a Google, recibe el código de autorización cuando el usuario regresa, lo intercambia por un token de sesión y lo almacena en el navegador. La aplicación puede consultar el estado de la sesión en cualquier momento usando \texttt{supabase.auth.getSession()}.

La base de datos es PostgreSQL, un sistema de bases de datos relacional de alto rendimiento. La comunicación se realiza mediante la biblioteca \texttt{supabase-js}, que permite ejecutar consultas directamente desde el navegador utilizando políticas de seguridad a nivel de fila (RLS) para controlar el acceso.

El servicio Realtime permite que los cambios en la base de datos se propaguen instantáneamente a todos los clientes conectados. UNDC Pets lo utiliza para que cuando un usuario crea, modifica o elimina una publicación, todos los demás usuarios que están viendo el muro vean el cambio sin necesidad de recargar la página.

\subsection{Cloudflare Pages}

Cloudflare Pages es la plataforma de alojamiento. Al ejecutar \texttt{npx wrangler pages deploy dist}, Wrangler sube los archivos de la carpeta \texttt{dist} a la red de Cloudflare, que los distribuye en servidores de todo el mundo. Cuando un usuario visita la aplicación, es redirigido al servidor más cercano geográficamente, minimizando el tiempo de carga.

Cloudflare Pages también maneja el enrutamiento de la SPA: como React maneja las rutas del lado del cliente, cualquier ruta que el usuario intente acceder directamente debe redirigirse a \texttt{index.html} para que React pueda interpretarla.

\section{Base de datos}

UNDC Pets utiliza cinco tablas en Supabase. A continuación se explica cada una, su estructura y su propósito.

\subsection{Tabla: posts}

Esta tabla almacena todas las publicaciones del muro de comunidad. Tiene tres columnas: \texttt{id} es un texto que funciona como identificador único, generado en el cliente con el formato \texttt{post\_} seguido de la marca de tiempo actual. \texttt{data} es una columna de tipo JSONB que contiene toda la información de la publicación como un objeto: autor, contenido, imagen, likes, comentarios, compartidos, etc. \texttt{created\_at} es una marca de tiempo que se asigna automáticamente al insertar la fila y se utiliza para ordenar las publicaciones por fecha.

La razón para usar JSONB en lugar de columnas separadas es que la estructura de las publicaciones es compleja y puede variar. Con JSONB, cada publicación puede tener exactamente los campos que necesita sin requerir cambios en el esquema de la base de datos.

\subsection{Tabla: pets}

Similar a la tabla de publicaciones, \texttt{pets} utiliza el mismo patrón de una columna \texttt{id} y una columna \texttt{data} JSONB. Cada fila representa una mascota del campus con toda su información: nombre, especie, género, edad, estado, ubicación, historia, foto y etiquetas.

Originalmente, las mascotas estaban definidas en un archivo JavaScript (\texttt{src/data.ts}) y se cargaban directamente en la aplicación. Para darle al administrador la capacidad de agregar, editar y eliminar mascotas sin modificar el código, se migraron a Supabase. Como mecanismo de respaldo, si la tabla no existe o está vacía, la aplicación utiliza los datos del archivo \texttt{data.ts} y mantiene una copia en el \texttt{localStorage} del navegador para que los cambios persistan incluso si la base de datos no está disponible.

\subsection{Tabla: admins}

Esta tabla es muy simple: solo contiene una columna \texttt{email} que es también la clave primaria. Cada fila representa un correo electrónico autorizado como administrador. Cuando un usuario inicia sesión, la aplicación consulta esta tabla para determinar si su correo está registrado. Si lo está, se activan las funcionalidades de administrador. Para añadir un nuevo administrador, solo es necesario insertar su correo en esta tabla.

\subsection{Tabla: donation\_config}

Esta tabla almacena toda la configuración relacionada con las donaciones en una sola fila. La columna \texttt{id} tiene el valor fijo \texttt{main} para identificar la fila única. La columna \texttt{data} es un objeto JSONB que contiene las cuentas bancarias (un arreglo con banco, número y CCI), los números de Yape y Plin, las rutas de los códigos QR para cada servicio y las campañas activas con sus montos y niveles de urgencia.

\subsection{Tabla: blocked\_users}

Cada fila en esta tabla representa un usuario bloqueado para publicar en la comunidad. La columna \texttt{user\_id} es el identificador único del usuario (proporcionado por Google al autenticarse), \texttt{reason} es el motivo del bloqueo, \texttt{blocked\_by} indica quién realizó el bloqueo (actualmente siempre \texttt{admin}) y \texttt{blocked\_at} es la fecha y hora del bloqueo. Cuando un usuario intenta crear una publicación, la aplicación verifica si su ID está en esta tabla antes de permitir la operación.

\section{Flujo de autenticación}

Cuando el usuario hace clic en ``Ingresar con Google'', ocurre la siguiente secuencia de eventos:

\begin{enumerate}
    \item El navegador redirige al usuario a la página de autenticación de Google.
    \item Google muestra la pantalla de selección de cuenta y solicita los permisos necesarios.
    \item Si el usuario acepta, Google genera un código de autorización y redirige al usuario de vuelta a la aplicación.
    \item Supabase intercepta esta redirección, intercambia el código por un token de sesión y lo almacena en el navegador.
    \item Supabase notifica a la aplicación mediante el evento \texttt{onAuthStateChange}.
    \item El componente \texttt{AuthProvider} en \texttt{AuthContext.tsx} detecta la sesión activa y consulta la tabla \texttt{admins} para verificar si el correo del usuario está registrado como administrador.
    \item La información (\texttt{user} y \texttt{isAdmin}) se pone a disposición de todos los componentes a través del contexto de React.
\end{enumerate}

Cada componente que necesita conocer al usuario actual (como \texttt{CommunityFeed}, \texttt{UserProfile} y \texttt{AdminPanel}) utiliza el hook \texttt{useAuth()} para obtener esta información. De esta manera, todos los componentes comparten la misma sesión sin necesidad de llamar individualmente a Supabase.

\section{Estructura del código fuente}

El código fuente de la aplicación se encuentra en la carpeta \texttt{src}, organizado de la siguiente manera:

\begin{itemize}
    \item \texttt{src/lib/}: módulos de infraestructura. \texttt{supabase.ts} crea y exporta el cliente de Supabase. \texttt{AuthContext.tsx} implementa el contexto de autenticación.
    \item \texttt{src/components/}: todos los componentes de la interfaz de usuario.
    \item \texttt{App.tsx}: componente raíz que orquesta toda la aplicación: maneja las pestañas de navegación, los estados globales, los filtros de búsqueda y la visualización de modales.
    \item \texttt{types.ts}: define todas las interfaces de TypeScript utilizadas en el proyecto, como \texttt{Pet}, \texttt{Post}, \texttt{Comment} y \texttt{DonationConfig}.
    \item \texttt{data.ts}: contiene los datos estáticos iniciales: las cuatro mascotas originales (Curly, Princesa, RunRun y Gata Ingeniera), publicaciones de ejemplo, preguntas frecuentes y puntos de acopio. Se utilizan como respaldo cuando la base de datos no está disponible.
\end{itemize}

\section{Despliegue}

El despliegue se realiza mediante Cloudflare Pages utilizando la herramienta Wrangler. El proceso consta de dos pasos.

Primero, construir la aplicación ejecutando \texttt{npm run build}. Este comando le indica a Vite que empaquete todo el código fuente, optimice los recursos y genere los archivos finales en la carpeta \texttt{dist}. Durante este proceso, Vite reemplaza las variables de entorno como \texttt{VITE\_SUPABASE\_URL} con sus valores reales, minimiza el JavaScript y el CSS, y aplica transformaciones para asegurar la compatibilidad con navegadores antiguos.

Segundo, subir la aplicación a Cloudflare ejecutando \texttt{npx wrangler pages deploy dist --project-name=undc-pets}. Wrangler sube la carpeta \texttt{dist} a Cloudflare, que la distribuye en su red global de servidores. El proyecto queda disponible en \texttt{https://undc-pets.pages.dev}.

Es importante configurar las variables de entorno en el panel de Cloudflare Pages. Las variables necesarias son \texttt{VITE\_SUPABASE\_URL} y \texttt{VITE\_SUPABASE\_ANON\_KEY}. Sin ellas, la aplicación no podrá conectarse a Supabase. También se puede configurar \texttt{VITE\_GEMINI\_API\_KEY} para habilitar el asistente de inteligencia artificial.

\section{Consideraciones de seguridad}

La seguridad en UNDC Pets se maneja en múltiples niveles:

\begin{itemize}
    \item \textbf{Autenticación:} Google OAuth garantiza que solo usuarios con cuentas de Google válidas accedan a las funciones interactivas. Supabase maneja la sesión de manera segura, almacenando los tokens en el navegador y renovándolos automáticamente cuando expiran.
    \item \textbf{Autorización:} La verificación de administrador se realiza consultando la tabla \texttt{admins}. Las operaciones críticas deberían idealmente estar protegidas por políticas RLS en Supabase para evitar que un usuario malintencionado modifique el código del cliente.
    \item \textbf{Datos:} La aplicación nunca almacena contraseñas ni información financiera sensible. Los datos de tarjeta ingresados en el formulario de donación son puramente simulados y no se envían a ningún servidor. Las claves de API de Supabase son claves anónimas diseñadas para uso desde el navegador, y las políticas RLS controlan las operaciones permitidas.
    \item \textbf{Contenido:} El sistema de bloqueo de usuarios permite al administrador impedir que usuarios problemáticos sigan publicando. Cuando un usuario bloqueado intenta crear una publicación, la aplicación detecta su ID en la tabla \texttt{blocked\_users} y rechaza la operación mostrando un mensaje de error.
\end{itemize}

\section{Sistema de tiempo real}

Una de las características más destacadas de UNDC Pets es la sincronización en tiempo real. Cuando un usuario crea, modifica o elimina una publicación, el cambio se refleja inmediatamente en todos los dispositivos que están viendo la página en ese momento, sin necesidad de recargar el navegador.

Esto se logra mediante Supabase Realtime, que utiliza la replicación lógica de PostgreSQL. Cuando se realiza un cambio en la tabla \texttt{posts}, PostgreSQL genera un evento que Supabase captura y reenvía a todos los clientes conectados a través de una conexión WebSocket persistente.

En el código, esto se implementa en \texttt{CommunityFeed.tsx} y \texttt{UserProfile.tsx} mediante el método \texttt{supabase.channel()}. Se crea un canal llamado \texttt{posts-changes} que escucha los eventos \texttt{INSERT}, \texttt{UPDATE} y \texttt{DELETE} en la tabla \texttt{posts}. Cuando llega un evento, el componente actualiza su estado local para reflejar el cambio.

Para que esto funcione, es necesario habilitar la replicación en la tabla \texttt{posts} ejecutando \texttt{ALTER PUBLICATION supabase\_realtime ADD TABLE posts;} en el editor SQL de Supabase.
